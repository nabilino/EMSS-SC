ujobdump - ujobload
Description
This section describes how to use the ujobdump - ujobload utilities. The Lawson ujobdump and ujobload utilities were created by Lawson to aid the customer when a new or updated screen (.scr) file has been placed in a productline as a result of CTP or MSP installation or an application upgrade. These utilities will update the job definitions and therefore, you do not have to manually change the job definitions when the .scr files are installed. Further information on all input parameters can be found at support.lawson.com

Note:  Only System Administrators can use this utility.
           DO NOT enter the lawson option parameters (-u, -t. -d) in any of the text areas.

Input Fields for ujobdump
  • Data Area:
    		•  Select a data area from drop down.
  • Verbose Option (-v):
    		•  Select the checkbox to list any jobs that are not dumped.
  • Job Count Only Option(-c):
    		•  Select the checkbox to displays the number of jobs that will be processed. This option does not process any jobs. Results are displayed on the screen.
  • List Jobs with Job Count Option (-l):
    		•  Select the checkbox to list all jobs that are counted with the Job Count Only (-c) option.
  • Include Upgrade Jobs Option (-x):
    		•  Select the checkbox to include upgrade jobs.
  • Output File:
    		•  Select the checkbox and enter the output file name. The actual file that will be create will be preceeded by ujob_(data area)_. This field is disabled when using the Job Counts Only (-c) option.
  • User Name [Job Names] (-u):
    		•  Select the checkbox and enter a user name to be dumped. If job names are listed after the user name, only those jobs for the given user will be dumped. Multiple jobs may be specied.
  • Backup Source Directory (-d):
    		•  Select the checkbox and select a backup (or alternate) source directory. The option specifies where backup source directories (acsrc, amsrc, apsrc, etc.) are located. They hold the original source for any programs that changed after applying an MSP or CTP and will cause ujobdump to only dump jobs associated with changed programs.
  • Program Tokens(-t):
    		•  Select the checkbox and enter token names to be dumped. If this option is specied, specic jobs cannot be entered with the -u option.

    Input Fields for ujobload
  • Data Area:
    		•  Select a data area from drop down.
  • Input File:
    		•  Select an Input File from the drop down.
  • Overwrite Option (-o):
    		•  Select the checkbox to overwrite existing jobs. If you do not specify this option, only jobs that do not exist will be created.
  • Update Option (-u):
    		•  Select the checkbox to update jobs. If you do not check the update option, no jobs will be overwritten or created.
  • Display Last Job Log Option (-l):
    		•  Select the checkbox to display the job log from the last ujobload run. You must select a product line. All other parameters are disabled when running this option.
  • Display Deleted Fields Option (-d):
    		•  Select the checkbox to list deleted fields.
  • Overwrite Job Names (-x):
    		•  Select the checkbox and enter translations. The translate option renames specied job names from OldJobName to NewJobName. You can specify as many translate options as necessary. For example: JOB-1:JOB-1a JOB2:JOB-2a
